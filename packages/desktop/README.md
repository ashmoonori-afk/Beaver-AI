# @beaver-ai/desktop

> Tauri v2 desktop shell wrapping the Phase 4U React UI. Replaces the terminal-prompt launcher with a native Win64 / macOS / Linux executable. (D17)

**Status:** 4D.1 (scaffold + Windows build) shipped. 4D.2 (real CLI sidecar wiring via `invoke()`) pending.

## Layout

```
packages/desktop/
├── package.json              # @beaver-ai/desktop — wraps Tauri CLI
└── src-tauri/                # Rust crate
    ├── Cargo.toml            # beaver-desktop binary
    ├── Cargo.lock            # tracked (binary app, reproducible builds)
    ├── build.rs
    ├── tauri.conf.json       # productName=Beaver, identifier=ai.beaver.desktop
    ├── capabilities/
    │   └── default.json      # core permission set for the main window
    ├── icons/                # generated by `tauri init`
    └── src/
        ├── main.rs           # entry — calls beaver_desktop_lib::run()
        └── lib.rs            # tauri::Builder + `desktop_info` command
```

## Build (production)

Prerequisites: **Rust 1.77+** (`rustup`) and **Node ≥ 22.6** (`pnpm`).

```bash
pnpm --filter @beaver-ai/desktop tauri build
```

Outputs (Windows host):

| Artifact        | Path                                                                      |
| --------------- | ------------------------------------------------------------------------- |
| Standalone .exe | `src-tauri/target/release/beaver-desktop.exe` (~9 MB)                     |
| MSI installer   | `src-tauri/target/release/bundle/msi/Beaver_0.1.0_x64_en-US.msi` (~3 MB)  |
| NSIS setup .exe | `src-tauri/target/release/bundle/nsis/Beaver_0.1.0_x64-setup.exe` (~2 MB) |

macOS / Linux produce `.dmg` / `.AppImage` respectively.

## How the launcher integrates

`Start-Beaver.bat` / `.command` / `.sh` at the repo root now:

1. Probe for the built `beaver-desktop` binary
2. If missing, run `pnpm --filter @beaver-ai/desktop tauri build` (one-off)
3. Launch the binary

Legacy CLI launchers (terminal prompt) preserved at `Start-Beaver-CLI.{bat,command,sh}` for headless / SSH workflows.

## IPC (Tauri commands)

4D.1 ships exactly one command — a smoke test:

| Command        | Returns                         | Purpose                                                                                     |
| -------------- | ------------------------------- | ------------------------------------------------------------------------------------------- |
| `desktop_info` | `{ version, target_os, debug }` | Confirm the IPC bridge is alive (renderer can call `invoke('desktop_info')` and round-trip) |

4D.2 will add (one per UI hook):

- `runs_start({ goal })` — spawns `node packages/cli/src/bin.ts run` as a sidecar; events stream back via `app.emit('run.snapshot', …)`
- `checkpoints_answer({ id, response })`
- `final_review_decide({ runId, decision })`
- `wiki_ask({ question })`

Until 4D.2 lands, the renderer continues to use the in-process mock transports from Phase 4U — clicking around the bento UI shows the demo, not real runs.
